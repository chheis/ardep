Register an incoming frame callback. When the device is a responder it will listen to frames with set frame_id. When the device is a commander it will be invoked when a schedule requests a frame and one of the responders answer with data.

◆ abstract_lin_schedule_now()

__syscall int abstract_lin_schedule_now ( const struct device *  dev,
uint8_t  frame_id 
)

Schedules to send a frame header over the lin bus and calls the outgoing or incoming callback accordingly.

When a outgoing callback with set frame_id is registered it gets invoked to first check whether to send the frame or not and to fill the frame with data.

When a incoming callback with set frame_id is registered a frame header is sent to request a respond from a responder.

Warning
Only usable by the commander.
Note
both receiving and sending are asynchronous if there is not currently data being sent/received. See lin_send() and lin_receive()
Parameters
devPointer to the Abstract LIN device
frame_idThe frame id to be scheduled to the bus
Return values
0On success
-EINVALif no outgoing callback is registered or if the frame_id is invalid
-ENOTSUPif the device is not a commander
othernegative error values from lin_send / lin_receive
